01 — The Problem

Three Broken Systems, One Root Cause

India's ₹7 lakh crore textile industry is dangerously exposed to monsoon disruptions — yet nobody had connected these dots with AI before RainLoom.

📉
Textile Stocks Crash Silently
When rainfall is 20%+ below normal, cotton yields drop. Input costs spike. Investors discover the crash only after it happens — too late to hedge.
No predictive system existed
🧑‍🌾
Farmers Wait 190 Days for Insurance
India's PMFBY crop insurance scheme requires inspector visits and mountains of paperwork. By the time a farmer gets paid, debt has compounded.
PMFBY avg claim: 190 days
👩
60% of Workers Are Invisible
60% of India's 45M textile workers are women. When factories shut due to cotton shortages, no public dataset shows who loses their income first.
Zero public gender-risk data

RainLoom fixes all three with a single causal AI pipeline: it reads the sky, proves causation mathematically, runs three ML models, and routes the output to investors, farmers, and policymakers — simultaneously.

Causal Proof› Expand
Prove It's Causation, Not Just Correlation
Before any ML model runs, RainLoom mathematically proves that monsoon causes textile stock volatility — using three independent statistical tests.
📊 Granger Causality
Lagged rain predicts stock vol?
Tests whether past rainfall values statistically improve predictions of future stock volatility. If yes — Granger says it's causal. Optimal lag: 4–6 weeks.
🎯 IV / 2SLS F=5.8
ENSO as exogenous instrument
ENSO drives monsoon but has NO direct effect on textile stocks — perfect instrument. First-stage F=5.8, p<0.05 confirms the causal chain.
🔗 VAR / Johansen
Long-run cointegration
Tests whether rainfall, cotton price, and stock volatility share a long-run equilibrium — proving the chain is structural, not coincidental.
⏱️ Total Lead Time
6–10 weeks early warning
Rainfall → cotton yield: 4–6w lag. Cotton price → stock vol: 2–4w lag. Combined: RainLoom warns investors 6–10 weeks before a crash.
# Granger Causality Test (causal.py) gc = grangercausalitytests(df[["stock_vol","rainfall"]], maxlag=12) # → p < 0.05 at lag 4 ✓ rainfall Granger-causes volatility # IV/2SLS: Stage 1 — Regress rainfall on ENSO # Stage 2 — Use fitted rainfall → stock volatility # F-stat 5.8 confirms instrument relevance

ML Ensemble› Expand
Three AI Models → One Risk Score (0–1)
MS-GARCH detects market regime, XGBoost classifies risk from 40+ features, MLP captures sequential patterns. Their weighted output = the Risk Score.
📉 MS-GARCH (regime.py)
Market mood detector
Detects CALM vs VOLATILE regime using return variance clustering. Outputs: regime probability + expected σ. Weight: 35%.
🌳 XGBoost (xgb_classifier.py)
Feature impact mapper
847 gradient-boosted trees on 40+ features. SHAP explainer shows exactly WHICH feature drove the risk. Weight: 40%.
🔡 MLP (lstm_model.py)
Sequential pattern finder
12-week sliding window neural net. Catches sustained drought trends that point-in-time models miss. Weight: 25%.
🎯 Final Score
AUC-ROC 0.81
0.0–0.4 = LOW · 0.4–0.6 = MODERATE · 0.6–0.8 = HIGH · 0.8+ = EXTREME. Every downstream action is triggered by this single float.
regime_prob = ms_garch.predict(returns) # volatility regime xgb_score = xgb_model.predict_proba(X)[1] # risk probability mlp_score = mlp.forward(X_window) # sequential score risk_score = 0.35*regime_prob + 0.40*xgb_score + 0.25*mlp_score # → single float [0,1] — drives every downstream decision

🌦️
Dual-Source Trigger Consensus ANTI-FRAUD
A payout can only fire if both independent data sources agree: IMD/Open-Meteo must report ≥ −20% rainfall deficit AND NASA NDVI must confirm crop stress ≤ 0.30. A false alert from one source alone is never enough to trigger payment — preventing fraud from data corruption or API spoofing.
🪪
Aadhaar eKYC + AEPS Biometric IDENTITY
Every registered farmer is authenticated via Aadhaar-linked eKYC (Government of India's biometric ID system). Payout is routed only to the verified bank account in the National Payments Council (NPCI) database — no cash, no third-party accounts. A fraudster cannot redirect funds without biometric authentication.
📍
District Geo-Fencing LOCATION
Farmers are enrolled with their land parcel's district code. At trigger time, the system checks that their registered district is among the 83 cotton-belt districts currently showing a validated rainfall deficit. A farmer in a non-affected district cannot claim — even if they are otherwise eligible.
📱
SMS OTP Confirmation 2FA
Before the UPI transfer is dispatched, a one-time password is sent to the farmer's registered mobile number. The farmer must confirm within 90 seconds. This two-factor step ensures that even if an Aadhaar number is compromised, money cannot move without phone access.
🔒
Per-Farmer Seasonal Payout Cap ABUSE PREVENTION
Each farmer account has a maximum of one payout per drought event per season. The event deduplication key is computed from: Farmer ID + District Code + Season Year + Drought Event ID. If the same key is seen again, the system silently marks it duplicate and blocks re-payment — preventing replay attacks or accidental double triggers.
📋
Immutable Audit Trail COMPLIANCE
Every payout event — including the exact rainfall value, NDVI reading, risk score, timestamp, farmer ID, district, and UPI transaction ID — is written to an append-only log. This log is the basis for regulatory reporting, dispute resolution, and actuarial review by insurance underwriters. Nothing can be silently modified.
